# PII Extraction Evaluation Results

## Objective
The objective of this project was to evaluate how well a large language model (LLM) can identify and extract personally identifiable information (PII) from text documents.

## Method
Sample documents containing PII were created and the LLM was prompted to extract structured PII fields including names, addresses, emails, phone numbers, employee IDs, reference numbers, dates of birth, organizations, and locations. The extracted data was evaluated for accuracy, completeness, and correctness.

## Findings
The AI model was generally able to identify common PII fields such as names, email addresses, phone numbers, and employee IDs. Some fields were partially extracted or missed depending on how the information was formatted in the text.

This demonstrates that AI systems can assist with PII detection and document review workflows, but human review is still necessary to ensure accuracy and completeness when handling sensitive personal data.

## Conclusion
LLMs can be useful tools for identifying and extracting PII for privacy, compliance, and data governance use cases. However, human oversight is important to ensure that all sensitive information is correctly identified and handled appropriately.
